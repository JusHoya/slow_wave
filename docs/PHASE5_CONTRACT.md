# Phase 5 — Shared Interface Contract (authoritative)

This file pins the cross-module interfaces for **Phase 5 (Experiments, Analysis
& Plots)** so independently-authored modules integrate without guessing. **Do not
deviate from these signatures.** Import shared Phase 0–4 types from their modules —
do not redefine them.

## Objective & exit criteria (PRD §8, Phase 5)
Execute the preregistered experiment grid in accelerated sim-time (≥5 seeds),
validate with a few real long-horizon runs, generate all figures, and record the
H1/H0 decision with its statistics. The work must satisfy, verbatim:

- **EC1** The full **arm × distractor-regime × seed** grid completes; every run
  has a manifest with **cost + git hash**; coverage (and any dropped cells) is
  logged — **no silent caps**.
- **EC2** **≥5 seeds per arm** (varying both seed types), with N consistent with
  the committed power analysis.
- **EC3** At least **one real long-horizon run per key arm** is executed, and a
  **sim-vs-real agreement** figure quantifies whether time-compression distorts
  retention curves (a documented inversion at scale counts as a finding).
- **EC4** Every figure is regenerated by `repro` from **committed manifests** in
  one command; each caption states **n** and the **CI method** (red-team re-runs
  it).
- **EC5** The **primary endpoint** is computed exactly as preregistered, with
  effect size + CI and the preregistered test; the H1/H0 verdict is stated
  unambiguously (confirmed / refuted / inconclusive-with-reason).
- **EC6** The **long-context crossover** (stream length L beyond which
  consolidation beats stuff-it-in-context, if any) is reported, or its absence
  stated.
- **EC7** If the result is **negative**, the analysis maps it to one of the
  preregistered negative forms (PRD §9.1) and ties it to the datasheet regime.

Maps to FR5.1–FR5.6, FR6.2 (figures from manifests), DX1 (determinism), DX2 (no
silent caps), DX4 (plots a reviewer trusts), DX5 (no overclaiming).

## Owner decisions locked for this sprint (do not re-litigate)
- **Substrate = the deterministic MOCK LLM** (this box has no `ANTHROPIC_API_KEY`,
  exactly as Phases 0–4). Every number is reproducible bit-for-bit. **The H1/H0
  verdict is therefore a MECHANISM DEMONSTRATION in the synthetic + mock regime —
  NOT a scientific claim about a real Claude model.** This caveat (DX5) is
  mandatory and must appear in the analysis (`AnalysisReport.mock_llm_caveat`),
  the `paper/RESULTS.md` writeup (top), and **every figure caption**.
- **8 seeds** for the full grid (above the prereg floor of 5; avoids the tied-A/A
  underpowering Phase 4 hit). The smoke grid uses 3 seeds for speed.
- **"Real long-horizon" = a long, low-compression stream**, contrasted with the
  **accelerated sim-time = short, high-compression stream** (PRD decision D4).
  This is NOT about real-vs-mock API — both use the mock LLM.

## Ground rules (inherited from Phase 0–4 — non-negotiable)
- Python **3.12** target (support 3.11+). Run tests with
  `.venv\Scripts\python.exe -m pytest`. The default `python` on this box is 3.14
  (too new for some wheels) — always use the venv interpreter.
- pydantic **v2**. `extra="forbid"` on every structured model. Reuse the shared
  Phase 0–4 types — **do not redefine them**: `Config`/`EvalConfig`,
  `Stream`/`AccuracyMatrix`/`ProbeSet`, `MemoryFootprint`, `ContinualMetrics`,
  `ExperimentResult`/`ArmResult`/`ArmCost`/`PruneQuality`/`PrimaryEndpoint`/

  `slow_wave.eval.phase5_schema` (already written by the orchestrator — consume

  `numpy.random.default_rng(derive_seed(seed, "<name>"))` or an explicitly-seeded
  generator — never Python `hash()` or the `np.random` global. Given a fixed
  config+seed under the mock LLM, every structured result is **byte-identical**
  across two runs. Only wall-clock timing (and, with a real Claude call, token
  counts / text) is nondeterministic — those fields are already flagged in the
  manifest's `nondeterministic_fields`.
- **LEAN CORE, NUMPY-ONLY STATS in module/library + test code.** CI installs only
  `.[dev]` = `pydantic`, `pyyaml`, `numpy`, `anthropic`, `pytest`. **`scipy`,
  `statsmodels`, `rliable`, `matplotlib`, `seaborn` are NOT in CI.** Therefore:
  - `slow_wave/eval/grid.py` and `slow_wave/eval/analysis.py` import **only**
    `numpy` (+ stdlib + pydantic + slow_wave). Reuse `slow_wave.eval.stats` for
    any CI/effect-size/test — it is already pure-numpy. **Never** `import scipy`/
    `statsmodels`/`rliable`/`matplotlib` in these modules or in their tests.

    `matplotlib`, and it must import it **lazily inside the function bodies**

    Its test (`tests/test_paper_figures.py`) MUST guard every matplotlib-touching

    where matplotlib is absent. matplotlib IS installed in this box's `.venv`
    (3.11.0), so figures actually render here for the red-team.
- **Confound guard (FR1.6) is sacred.** Only the **`oracle`** arm and **offline
  scoring** (`prune_metrics`, the probe scorer) may read ground-truth labels, and
  only via `slow_wave.stream.schema.offline_labels`. Phase 5 code reads labels
  **only** transitively through the already-built `ArmResult.prune_quality` /
  `ExperimentResult`; never read `offline_labels`/`ground_truth`/`probe.answer`
  directly in grid/analysis/figures.
- **Honesty by construction (DX2).** Anything bounded (a dropped cell, a sweep
  cap, an unmatched budget, a top-N) is recorded in a result field **and**
  `logger.info/warning`-logged. Never drop silently.
- **No overclaiming (DX5).** See the mock-LLM caveat above. Frame neuroscience as
  motivation, never as efficacy/biological-fidelity proof. The TMR figure is a
  replay-targeting **analogue/proxy**, not a literal cued-TMR protocol — say so.
- **Import by full module path** (`from slow_wave.eval.grid import run_phase5`),
  NOT via the package root.
- Google-style docstrings on every public function/class, matching the density
  and tone of the existing Phase 0–4 modules.

## WS1 — Sweeps & grid runner (`slow_wave/eval/grid.py` + configs)

### Public API
```python
def run_regime_sweep(cfg: Config, out_dir: Path) -> GridResult:
    """Run the arm × distractor-regime × seed grid (EC1).
    For each regime in `cfg`-derived regimes (>= 3; see config `phase5.regimes`),
    deep-copy cfg, set cfg.stream.label_mix to the regime, run build_experiment,
    write the per-cell ExperimentResult manifest to
    <out>/phase5/regime_<name>/manifest.json (reuse run_experiment's manifest
    writer or new_manifest+write_manifest), and roll the summary arrays into a
    RegimeCell (per-arm per-seed acc / prune_precision / prune_recall / prune_f1 /
    signal_retention / total_tokens / memory_vectors; the FULL `primary_endpoint`
    object copied from the cell's ExperimentResult + the mirrored scalar
    value/verdict/CI fields; the A/A floor; budget_matched). primary_regime = the
    registered distractor-heavy regime. Log coverage (DX2: 'ran R regimes x A arms
    x S seeds = N cells; 0 dropped')."""

def run_length_sweep(cfg: Config, out_dir: Path,
                     *, lengths: list[int], key_arms: list[str]) -> LengthSweep:
    """Stream-length sweep feeding the long-context crossover (EC6 data).
    For each L in lengths and each seed in cfg.eval.seeds, generate ONE stream of
    n_tasks=L (deep-copied cfg.stream), build probe set, and run each key_arm via
    run_arm; record per-(arm,seed) ACC, total_tokens, memory_vectors into a
    LengthPoint. key_arms default ['full_dream','long_context','no_sleep']. Log
    coverage."""

def run_sim_vs_real(cfg: Config, out_dir: Path, *, key_arms: list[str],
                    sim_n_tasks: int, sim_compression: float,
                    real_n_tasks: int, real_compression: float) -> SimRealResult:
    """Accelerated-sim vs real-long-horizon runs per key arm (EC3).
    Build a 'sim' cfg (n_tasks=sim_n_tasks, sim_time.compression_factor=
    sim_compression) and a 'real' cfg (n_tasks=real_n_tasks, compression=
    real_compression); for each key_arm run every seed via run_arm on both,
    collecting acc + the final-row retention curve (RetentionCurve). Compute
    pearson_agreement & spearman_agreement between the per-arm mean-ACC vectors
    (sim vs real), ranking_preserved + inversions, max_abs_acc_divergence, and an
    honest `note`. key_arms default ['full_dream','no_sleep','long_context'].
    Use numpy for the correlations (np.corrcoef; rank via argsort for spearman).
    Single arm / zero-variance -> agreement defined as 1.0 with a note (never
    raise)."""

def run_phase5(cfg: Config, out_dir: str | Path | None = None) -> Path:
    """One command: run all three sweeps, assemble a Phase5Result (analysis=None),
    write it to <out>/phase5/phase5_result.json (pretty, sort_keys), and return
    the path. Populate git_commit from git_info(), model_id/model_mocked from cfg
    + a probe LLM result (mock => True), retention[primary_regime] = per-arm
    RetentionCurve from the primary regime cell's manifest, manifest_paths = every
    per-cell manifest written, coverage_notes spanning all sweeps. CLI:
    `python -m slow_wave.eval.grid --config configs/phase5_smoke.yaml`
    (argparse --config/--out, mirror eval/runner.py)."""
```
### Config shape (new `phase5:` block on the top-level config OR a sibling YAML)

## WS2 — Analysis & verdict (`slow_wave/eval/analysis.py`)
Owns `slow_wave/eval/analysis.py`, `tests/test_eval_analysis.py`. **numpy +
stdlib + pydantic + slow_wave only** (reuse `slow_wave.eval.stats`). Reads a

### Public API
```python
def analyze(result: Phase5Result, *, prereg_path: str =
            "prereg/preregistration.yaml", stats_seed: int = 0) -> AnalysisReport:
    """Compute the Phase 5 analysis verdicts (EC5–EC7) from a Phase5Result.
    PRIMARY (EC5): take the registered primary regime's RegimeCell; the primary
    verdict/value/CI/test/effect/noise_floor are surfaced from
    `cell.primary_endpoint` (the FULL PrimaryEndpoint object WS1 embedded,
    computed by the harness exactly as preregistered). You MUST still assert the
    endpoint NAME against the prereg via load_preregistration(prereg_path) +
    assert_primary_endpoint(prereg, cell.primary_endpoint.name) (DX3: refuse a
    non-preregistered endpoint). CI method string = cell.primary_endpoint
    .difference_ci.method. (If cell.primary_endpoint is None, fall back to loading
    the per-cell manifest at cell.manifest_path and reading
    .results['experiment']['primary_endpoint']; if still absent, verdict =
    'not-computed' with a coverage note.)
    CROSSOVER (EC6): from result.length_sweep, compute acc_per_token per arm per L
    (mean ACC / mean total_tokens; guard divide-by-zero), find the smallest L
    where treatment acc_per_token > baseline; also the raw-ACC crossover; fill a
    CrossoverResult with an honest `note` (found at L=… OR 'no crossover in swept
    range [..]; long_context dominates on raw accuracy at max cost' ).
    TMR (FR5.3): pool signal_retention for replay arms {full_dream, replay_only,
    reflection} vs no-replay {no_sleep, downscale_only} from the primary regime
    cell; hedges_g = bias-corrected unpaired Cohen's d (use stats.cohens_d
    paired=False then * (1 - 3/(4*(n1+n2)-9))); bootstrap CI via
    stats.bootstrap_ci on the per-resample g OR reuse effect_size_cohens_d CI as a
    documented approximation; benchmark_g=0.29.
    POWER (EC2): n_seeds=len(seeds); observed_effect_d=|primary effect d|;
    required_n_for_observed via Colas et al. 2018 paired formula
    n = ceil(((z_{0.975}+z_{0.8})^2)/d^2)+1 with z_{0.975}=1.959964, z_{0.8}=
    0.841621 (guard d->0 => a large sentinel like 9999 + note); floor=5.
    NEGATIVE (EC7): applicable = primary_verdict != 'confirmed' OR any registered
    secondary negative observed. ALWAYS compute the secondary contrast
    'full_dream vs replay_only' (PRD §9.1#1) from the primary cell's per-seed ACC
    via stats.paired_diff_ci + wilcoxon; matched_forms drawn from
    prereg.negative_result_forms; regime_tie = the primary regime name + mix.
    Fill per_regime_verdicts from every cell, sim_real_agreement_note from
    result.sim_real.note, a plain-language `headline`, and the mandatory
    `mock_llm_caveat`. Deterministic given stats_seed."""

def write_analysis(result: Phase5Result, report: AnalysisReport, out_dir: Path
                   ) -> tuple[Path, Path]:
    """Write <out>/phase5/analysis.json (report.model_dump json, sorted) and
    paper/RESULTS.md (a human-readable writeup generated FROM `report`), and also
    rewrite phase5_result.json with result.analysis = report. Return both paths.
    RESULTS.md MUST open with the mock-LLM caveat, then: the primary verdict
    (value, CI + method, test + p, effect + magnitude, noise floor), per-regime
    verdict table, the crossover statement (EC6), the TMR effect vs 0.29, the
    realized-power statement (EC2), the sim-vs-real statement (EC3), and the
    negative-result mapping (EC7)."""

def run_analysis(result_path: str | Path, *, prereg_path=..., out_dir=...) -> Path:
    """CLI helper: load Phase5Result from result_path, analyze, write, return the
    analysis.json path. CLI `python -m slow_wave.eval.analysis --result
    runs/phase5/phase5_result.json` (argparse)."""
```
### Tests (`tests/test_eval_analysis.py`) must cover
- `analyze` on a hand-built `Phase5Result` where the primary regime cell has
  full_dream strictly > no_sleep per seed yields `primary_verdict` matching the
  cell's recorded verdict and an effect with the right sign; on equal arms yields
  a non-confirmed verdict with a CI spanning 0.
- The endpoint name is asserted against the prereg: feeding a mismatched
  requested name (if the API exposes it) raises `NonPreregisteredEndpointError`
  (DX3) — at minimum assert `analyze` uses the prereg's `primary_endpoint`.
- Crossover: on a synthetic `LengthSweep` where treatment acc_per_token overtakes
  baseline at a known L, `crossover_length` equals it and `crossover_found` True;
  where it never overtakes, `crossover_found` False and `note` states absence.
- TMR: replay arms with higher signal_retention than no-replay yield
  `hedges_g > 0`, finite CI, and `exceeds_benchmark` set correctly vs 0.29.
- Power: a large observed d yields `required_n_for_observed` small and
  `powered_for_observed` True; a tiny d yields a large required n and a note.
- Negative mapping: a refuted primary yields `applicable True` with >=1
  matched_form and the regime tie; the secondary `full_dream vs replay_only`
  contrast is always populated.
- `write_analysis` writes both files; `RESULTS.md` contains the mock-LLM caveat
  and the words "primary endpoint", "crossover", "TMR", "power"; `analysis.json`
  round-trips as `AnalysisReport`; determinism (byte-identical on re-run).

## Integration (orchestrator-owned — do NOT implement in a workstream)
After the three workstreams land, the orchestrator: runs `run_phase5` on the full
config (8 seeds), runs `run_analysis` on the result, runs `generate_all_figures`,
adds the `repro-phase5` Makefile target + a one-command note, writes the README/
contract notes and `tests/test_eval_phase5_integration.py` (the end-to-end
smoke: run_phase5 → run_analysis → generate_all_figures on the smoke config,
asserting every exit criterion's artifact exists and the verdict is stated),
decides what is small enough to commit (the `phase5_result.json` + `analysis.json`
+ figures + smoke manifests are the committed artifacts; oversized full per-cell
manifests, if any, are regenerable and `.gitignore`d with a note). Keep modules
import-clean so this glue is mechanical.
